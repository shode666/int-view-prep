# บทที่ 19 — Story Bank, Intro Pitch และคำถามย้อนถาม

> บทเดียวในเล่มที่คุณต้องเขียนเองครึ่งหนึ่ง — template พร้อมแล้ว เติมเรื่องจริงของคุณลงไป แล้วซ้อมเล่าออกเสียง

## เข็มทิศก่อนอ่าน

ถ้าบท 17 และ 18 สอนรูปแบบการตอบ บทนี้คือคลังกระสุนที่จะทำให้การตอบของคุณมีเนื้อจริง Story bank ที่ดีช่วยให้คุณไม่ต้องด้นสดทุกครั้ง และไม่ต้องหวังว่าความทรงจำจะทำงานดีภายใต้แรงกดในห้องสัมภาษณ์

อ่านบทนี้แบบลงมือเขียนตามจริง เพราะถ้าไม่เติมเรื่องของตัวเอง บทนี้จะไม่มีประโยชน์เกินกว่าการอ่าน template ผ่าน ๆ เป้าหมายคือให้หนึ่งเรื่องตอบได้หลายมุมและเรียกใช้ได้ทันที

## Story Bank — เตรียม 6–8 เรื่อง ใช้ตอบได้ทุกคำถาม

เรื่องหนึ่งใช้ตอบได้หลายคำถาม — เตรียมให้ครอบ 8 หมวดนี้ (เรื่องเดียวติ๊กได้หลายช่อง):

| # | หมวด | คำถามที่มันตอบได้ |
|---|---|---|
| 1 | Production incident หนักๆ | failure, pressure, ownership, postmortem |
| 2 | Conflict เชิง technical ที่แพ้หรือเสมอ | conflict, feedback, ego |
| 3 | ผลักดัน tech debt / refactor ใหญ่สำเร็จ | ต่อรอง business, ริเริ่ม, ผลระยะยาว |
| 4 | สร้าง/พัฒนาคนจนโตชัดเจน | mentoring, delegation, สเกลทีม |
| 5 | ตัดสินใจ architecture ที่ **ผิด** แล้วแก้ | judgment, humility, เรียนรู้ |
| 6 | ส่งของภายใต้ deadline โหด | prioritization, scope negotiation |
| 7 | เปลี่ยน process ของทีม (CI, review, on-call) | leadership without authority |
| 8 | งานที่ภูมิใจสุดเชิง technical | ใช้เปิด "tell me about yourself" ต่อได้ |

### Template ต่อเรื่อง (เติมเองทุกช่อง)

```
ชื่อเรื่อง (จำง่ายๆ): ....................
S — บริบท (2 ประโยค): ระบบอะไร ทีมกี่คน เดิมพันคืออะไร
T — หน้าที่ผม: ....................
A — ผมทำอะไร (3–5 ขั้น, ใช้คำว่า "ผม"): 
   1. ....................
   2. ....................
   3. ....................
R — ผลลัพธ์ (ต้องมีตัวเลข ≥ 1 ตัว): ....................
เรียนรู้ / ทำต่างไปหลังจากนั้น: ....................
ใช้ตอบหมวด: [ ]incident [ ]conflict [ ]debt [ ]mentor [ ]ผิดพลาด [ ]deadline [ ]process
เวลาเล่า: ต้องจบใน 2–3 นาที (ซ้อมจับเวลาแล้ว: ......)
```

เคล็ดที่ทำให้เรื่องดูจริง: มี **จุดที่คุณลังเลหรือพลาด** อยู่ในเรื่อง + มีชื่อเทคโนโลยี/ตัวเลขเฉพาะเจาะจง (ไม่ใช่ "ระบบใหญ่มาก user เยอะมาก")

## "Tell me about yourself" — pitch 90 วินาที

โครงสามท่อน (อดีต → ปัจจุบัน → ทำไมที่นี่):

1. **อดีต (30 วิ)**: เส้นทางย่อ + จุดแข็งที่สะสมมา — "ผมเป็น developer มา X ปี สาย backend/fullstack, ช่วงหลังโฟกัส Y, เคยดูแลระบบ Z ที่..."
2. **ปัจจุบัน (30 วิ)**: บทบาทตอนนี้ + ความสำเร็จหนึ่งอย่างที่วัดได้ + ส่วนที่เป็น lead (คน/กระบวนการ ไม่ใช่แค่ code)
3. **ทำไมที่นี่ (30 วิ)**: สิ่งที่อยากทำต่อ ↔ ตรงกับตำแหน่งนี้ยังไง — ต้อง **research บริษัทมาก่อน** พูดชื่อ product/challenge ของเขาได้

กติกา: ห้ามไล่ Curriculum Vitae (CV) ทีละบรรทัด (เขาอ่านมาแล้ว), จบด้วยประเด็นที่อยากให้เขาถามต่อ (เกี่ยวเรื่องที่คุณแน่นสุด), ซ้อมจนพูดได้เป็นธรรมชาติไม่ใช่ท่องบท

## คำถามย้อนถาม Interviewer

การไม่ถามอะไรเลย = red flag / คำถามดี = ข้อมูลตัดสินใจของคุณเอง + คะแนนปิดท้าย

**ถาม Engineering Manager / ว่าที่หัวหน้า:**

- "ความท้าทายใหญ่สุดของทีมตอนนี้คืออะไร ทั้งเชิงเทคนิคและเชิงคน"
- "คนที่ทำตำแหน่งนี้ได้ดีมากๆ ใน 1 ปี หน้าตาความสำเร็จเป็นยังไง" (ได้ job description จริงที่ไม่ได้เขียนไว้)
- "ทำไมตำแหน่งนี้ว่าง" (โตขึ้น? ลาออก? ทีมขยาย? — คำตอบบอกสุขภาพองค์กร)

**ถามเรื่องทีมและวัฒนธรรม:**

- "process ตั้งแต่ commit ถึง production เป็นยังไง ใช้เวลาเท่าไหร่" (วัด maturity ได้แม่นกว่าถามว่า 'agile ไหม')
- "incident ล่าสุดจัดการกันยังไง มี postmortem ไหม" (วัฒนธรรม blame หรือ blameless — สำคัญมากกับตำแหน่ง lead)
- "ทีมมีอำนาจตัดสินใจ technical แค่ไหน อะไรต้องขอใคร"

**ถามเรื่อง scope ของตำแหน่ง (สำหรับ lead):**

- "ตำแหน่งนี้แบ่งเวลา coding กับ management ประมาณไหน" (expectation mismatch ตรงนี้คือสาเหตุลาออกอันดับต้น)
- "งบ headcount ปีนี้เป็นยังไง ทีมจะโตไหม"

หลีกเลี่ยงในรอบแรก: เงินเดือน/สวัสดิการ (ไว้รอบ Human Resources (HR)/offer), คำถามที่ google ได้ใน 10 วินาที

## เช็คลิสต์ก่อนวันสัมภาษณ์

- [ ] Story bank ครบ 6–8 เรื่อง ซ้อมออกเสียงแล้วทุกเรื่อง จับเวลาแล้ว
- [ ] Pitch 90 วินาที ซ้อมแล้ว
- [ ] Research บริษัท: product, tech stack (จาก Job Description (JD)/blog/GitHub), ข่าวล่าสุด, คนที่จะสัมภาษณ์ (LinkedIn)
- [ ] เตรียมคำถามย้อนถาม 5 ข้อ (เผื่อบางข้อถูกตอบไปแล้วระหว่างคุย)
- [ ] ทวนบท 16 (checklist) + บท 11 (5 คำถามคลาสสิก) + บท 18 (framework) เช้าวันสัมภาษณ์
- [ ] ซ้อม pair programming ในโปรเจกต์นี้ (`pnpm test:watch`) — คิดออกเสียงระหว่างเขียน

บทต่อไป (บทสุดท้าย): DevOps & Security essentials — ของที่ lead โดนถามแต่ dev มักไม่ได้เตรียม

## สรุปท้ายบท

- story bank ที่ดีช่วยเปลี่ยนประสบการณ์กระจัดกระจายให้กลายเป็นคำตอบที่หยิบใช้ได้หลายสถานการณ์
- intro pitch และคำถามย้อนถามไม่ใช่ของแถม แต่เป็นส่วนหนึ่งของการกำหนดภาพจำที่ interviewer จะมีต่อคุณ
- การเตรียมเรื่องเล่าล่วงหน้าช่วยลดการด้นสดภายใต้แรงกดและทำให้คำตอบมีโครง มีตัวเลข และมีบทเรียนชัดขึ้น
- บทนี้จึงเป็นสะพานจาก "มีประสบการณ์" ไปสู่ "เล่าประสบการณ์นั้นได้อย่างน่าเชื่อถือ"

## ก่อนไปบทถัดไป

ก่อนปิดเล่ม บทถัดไปจะเก็บอีกหมวดที่ Dev Lead ถูกถามบ่อยแต่หลายคนเตรียมไม่พอ คือ DevOps และ security essentials ที่เกี่ยวกับการพาของขึ้น production อย่างปลอดภัย
